# TruthKeep v11 Runtime Profiles / Performance Modes

TruthKeep's core direction is fixed: **remember what is still true**. Runtime Profiles do not replace the core. They only decide when optional heavy math/crypto simulators are allowed to run.

## Core DNA that never turns off

Every runtime profile keeps these enabled:

- Graph governance
- Memory correction
- Superseded suppression
- Why-not provenance
- Invariant enforcement

This protects the project from turning into an unconstrained associative memory system. TruthKeep remains correctness-first.

## Profiles

| Profile | Purpose | Heavy engines in hot path | Recommended use |
|---|---|---:|---|
| `demo` | Small demos/tests | No | Quick demos, weak machines |
| `local` | Default local MCP usage | No | OpenClaw/Cursor/Claude local memory |
| `hardened` | Strict privacy/security candidate | Allowed when requested | Security experiments, strict local privacy |
| `enterprise` | Throughput-oriented runtime | No | Many agents/scopes, larger datasets |

## Commands

```bash
truthkeep profile show
truthkeep profile list
truthkeep profile set local
truthkeep profile set hardened
truthkeep profile doctor
truthkeep profile clear
```

Environment variable override:

```bash
TK_RUNTIME_PROFILE=enterprise truthkeep mcp-probe
```

MCP host configs generated by `truthkeep mcp install <host>` default to:

```json
{
  "env": {
    "TK_RUNTIME_PROFILE": "local",
    "TRUTHKEEP_TRANSPORT": "stdio",
    "TRUTHKEEP_OPEN_PORTS": "0"
  }
}
```

## Hot path vs cold path

Hot path commands are used constantly by agents:

- `memory_remember`
- `memory_recall`
- `memory_correct`

Cold path tasks are diagnostic or research-oriented:

- FHE/PQC simulators
- TDA signatures
- benchmark reports
- heavy audits

The `local` and `enterprise` profiles keep heavy simulators out of hot-path MCP usage so TruthKeep stays responsive.

## Benchmarking profiles

```bash
truthkeep benchmark profiles --records 10
```

This writes:

- `reports/profile_benchmark.json`
- `reports/profile_benchmark.md`

The benchmark is a local regression guard, not a cloud load test.
